**课程名称**：编译原理与技术实践
**实验日期**：2026年01月13日

实践亮点：
1. 在 LR 分析器 实现中，maker.cpp自动生成 FIRST/FOLLOW 集和 LR 分析表，并导出为头文件LRTable.h，消除运行时建表开销，且无需人工计算分析表。(3)
2. 在 语法树可视化 模块中，实现了自动生成AST的Graphviz DOT 语言与图片导出，直观对比LL与LR的建树差异。(2、3)
3. 在 符号管理 模块中，对不同阶段选择了最佳实践的数据结构（红黑树、Trie + 哈希表双向映射、二维数组、，实现 O(1)到O(logn)级的增删改查(4)。(1、2、3、4)
4. 在 词法与语法分析 阶段，采用了 **确定性有限自动机 (DFA)** 与 **表驱动架构**：词法分析器通过 DFA 状态迁移实现复杂 Token 识别(1)，LL 语法分析器依托预测分析表实现无回溯的自顶向下解析(2)，LR 语法分析器依托 ACTION/GOTO 分析表实现自底向上的移进-规约解析(3)。(1、2、3)
5. 在 语义分析 阶段，采用了 解释器+中间代码生成并行，边执行边生成四元式，通过符号表统一维护运行时状态，实现了变量类型自动提升，并支持符号表快照（CSV）导出。(4)
6. 在 错误处理环节 中，设计了 **多层次错误恢复机制**：语法层采用了恐慌模式、虚拟插入与前瞻验证(2、3)；语义层实现了 类型检查与运行时错误检测(4)。(2、3、4)
7. 在 错误定位环节 中，设计了 **精确行号追踪机制**，引入`lastAcceptedTokenLine` 等，在存在空行或跨行语句时仍能将错误定位到实际出错语句行。(2、3)
8. 在 测试与验证 中，构造了覆盖正确和多类错误用例的测试集，编写了**自动化测试流水线** `run_tests.py`，实现四个模块的一键编译、批量测试和结果归档。(1、2、3、4)
9. 在 项目架构 设计上，采用高内聚低耦合的模块化结构：LR 分析表由生成器 `maker.cpp` 产生，解析器仅消费表数据，形成典型生产者-消费者模式；可视化模块独立封装，可被 LL/LR 复用；各阶段通过统一接口串联成可扩展的编译流水线。(1、2、3、4)
10. 在 项目管理 中，使用 Git 托管到 GitHub 仓库（https://github.com/ying-2626/Principles-of-Compilation），便于版本管理。(1、2、3、4)
